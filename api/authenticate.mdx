---
title: "Authenticate"
description: "Authenticates the player session and retrieves balance, bet configuration, and jurisdiction flags."
---

## Signature

```typescript
client.Authenticate(): Promise<AuthenticateResponse>
```

`Authenticate` must be the first method called on a new client. It posts the `sessionID` and language to the RGS `/wallet/authenticate` endpoint, then stores the returned configuration and balance on the client for use by subsequent calls.

<Warning>
  All other client methods — `Play()`, `EndRound()`, and `Event()` — throw if `Authenticate()` has not been called first.
</Warning>

## What it does

1. POSTs `{ sessionID, language }` to `/wallet/authenticate`.
2. Stores `config` (bet levels, limits) and `jurisdictionFlags` on the client.
3. Parses the raw balance and emits a `balanceUpdate` window event.
4. If the server returns an active round from a previous session, emits a `roundActive` window event with `{ active: true }` and sets the internal `roundActive` flag.
5. Returns the full `AuthenticateResponse`.

## Side effects

| Event | Condition |
| --- | --- |
| `window` emits `balanceUpdate` | Always, with the parsed `Balance` as event detail. |
| `window` emits `roundActive` | Only when `round.active === true`, with `{ active: true }` as event detail. |

## Throws

Throws an `Error` if the RGS API returns a non-2xx HTTP status.

## Example

```typescript
import { RGSClient } from 'stake-engine';

const client = RGSClient({ url: window.location.href });

const { balance, config, jurisdictionFlags, round } = await client.Authenticate();

// Configure the bet selector using the returned bet levels.
const defaultBet = config.betLevels[config.defaultBetLevel];
console.log('Starting bet:', defaultBet);

// Respect jurisdiction flags.
if (jurisdictionFlags.disabledAutoplay) {
  hideAutoplayButton();
}

// Resume any interrupted round from a previous session.
if (round?.active) {
  console.log('Resuming active round', round.betID);
  await resumeRound(round);
}
```

<Note>
  Listen for `balanceUpdate` window events to keep your UI in sync with balance changes across all client methods, not just `Authenticate()`.
</Note>
